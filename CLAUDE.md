# MJBlount — Client Analytics Dashboard

## Project overview
A private agency dashboard for MJBlount Web Design. This is for the agency owner's eyes only — there is no client-facing view of the dashboard itself. Clients receive branded PDF reports exported from the dashboard, not access to the dashboard.

Each client has a fully branded dashboard page driven by their JSON config file.

---

## Tech stack
- **Framework:** Next.js (App Router)
- **Styling:** Tailwind CSS
- **Data source:** `/data/clients/*.json` — one file per client
- **Analytics:** Google Analytics 4 API (per client GA4 property ID)
- **AI summaries:** Anthropic API (claude-sonnet-4-20250514) — cached, not generated on every load
- **PDF export:** Puppeteer or React-PDF for branded client reports
- **Hosting:** Vercel or Netlify

## Dashboard page layout

The client dashboard (`/dashboard/[slug]`) is structured as follows, top to bottom:

### 1. Top bar
- Client logo + name + domain
- "Last updated" timestamp
- Alert badge if any modules have issues
- "Export report" button → navigates to `/dashboard/[slug]/export`

### 2. AI overview
- Plain-English summary generated by Claude
- Cached — generated once per data refresh, not on every page load
- Shows "last generated" timestamp + manual "Refresh" button
- Cache stored in `/data/cache/[slug]-summary.json`

### 3. Module overview grid (2-column)
Six module cards, each showing:
- Module name + status pill (Good / Issues / Review)
- Key metric or top finding
- Coloured left border: green = good, amber = needs attention
- Click → opens slide-out panel with full detail

Modules in order:
1. Traffic & behaviour (GA4)
2. ROI calculator
3. Technical audit
4. Site speed
5. SEO / AEO audit
6. Strategic audit

### 4. Site maintenance section
Always visible. No toggle, no "my eyes only" UI.
Shows: WordPress version, PHP version, SSL expiry, plugin update status.
Data source: WP REST API or manual input in client JSON.

## Module detail — slide-out panel
Clicking any module card opens a `<SlidePanel>` component from the right.
- Overlays the dashboard (does not navigate away)
- Has a close button and back-to-dashboard link
- Contains the full breakdown for that module
- Each panel is a separate component in `/components/dashboard/[module]/`

## ROI calculator
Inputs (stored in client JSON under `roi` key):
- `conversion_types`: array of { name, monthly_count (from GA4), value_per_conversion }
- `close_rate`: percentage (e.g. 0.6 for 60%)
- `build_cost`: one-off build fee
- `amortise_months`: months to amortise build cost over (default 12)
- `monthly_retainer`: ongoing fee

Outputs (calculated client-side, not stored):
- Estimated revenue = sum of (count × close_rate × value) per conversion type
- Cost per lead = monthly_cost ÷ total enquiries
- ROI multiplier = estimated_revenue ÷ monthly_cost
- Verdict: dynamically generated string, colour-coded green/amber/red

Inputs are editable in the dashboard UI and saved back to the client JSON via a PATCH API route.

## AI summary caching
```typescript
// lib/cache.ts
export async function getCachedSummary(slug: string): Promise<CachedSummary | null> {
  const cachePath = path.join(process.cwd(), 'data/cache', `${slug}-summary.json`)
  if (!fs.existsSync(cachePath)) return null
  const cache = JSON.parse(fs.readFileSync(cachePath, 'utf-8'))
  return cache
}

export async function writeSummaryCache(slug: string, summary: string) {
  const cache = { summary, generatedAt: new Date().toISOString() }
  fs.writeFileSync(
    path.join(process.cwd(), 'data/cache', `${slug}-summary.json`),
    JSON.stringify(cache, null, 2)
  )
}
```

## Report export
Route: `/dashboard/[slug]/export`

The report builder lets you:
1. Select which modules to include (checkboxes)
2. Choose date range
3. Preview the branded output
4. Export as PDF

The exported PDF uses the client's brand from their JSON:
- Their logo (appropriate variant for light background)
- Their primary and accent colours
- Their font (Karla Bold for web-safe fallback)
- Auto-generated date and "Prepared by MJBlount Web Design" footer

No client-specific values (your fees, close rate assumptions) are included in exports unless explicitly toggled on per module.

## Client data system
Each client is defined by a JSON file in `/data/clients/[slug].json`. This is the single source of truth for:
- Brand colours, fonts, logo variants
- GA4 property ID and GTM container ID
- Business info, services, tone of voice
- ROI calculator inputs and conversion type definitions
- Site maintenance metadata

**Never hardcode client-specific values in components.** Always read from the client JSON via `getClient(slug)`.

---

## Adding a new client
1. Create `/data/clients/[new-slug].json` using `back-garden-pizzeria.json` as the template
2. Add logo files to `/public/clients/[new-slug]/`
3. The route `/dashboard/[new-slug]` works automatically
4. No other code changes needed

---

## Coding conventions
- TypeScript throughout
- Tailwind for all styling
- No hardcoded hex values in components — always pull from `client.brand`
- Keep components generic and client-agnostic
- All GA4 and external API calls are server-side only
- ROI calculations are client-side (reactive, no server needed)
- AI generation is server-side, result cached to disk
